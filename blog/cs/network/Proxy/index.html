Proxy

thunder
개발자

크롤링할 때 IP를 바꿔야 할 때도 사용하고, Spring Boot로 개발할 때도 영속성 관련해서 Proxy를 공부한다. Nginx에서 리버스 프록시다 뭐다 하며 요청을 정리해 줄 때도 쓰는 용어...

제대로 알고 쓰자.

Proxy란?​

Proxy를 구글 번역기에 입력하면 "대리" 라고 나온다. 단 한 줄로 프록시를 정의할 수 있을까?

  • 대신 해주는 중간자라고 표현하면 어떨까 싶다.

GPT의 두 줄 대답:

Proxy는 요청을 직접 대상에게 보내지 않고,
중간에서 대신 받아 처리하거나 전달하는 존재다.

실제 겪은 예시들​

  1. 데이터를 크롤링할 때
내 PC -> Proxy Server -> 데이터 긁어오는 사이트
  • 특정 IP에서 과도하게 접속하면 서버에 부하를 줄 수 있어 차단당할 수 있는데, 이때 프록시를 통해 접속 경로를 분산(오해의 소지)
  1. Nginx에서 80번 포트에서 받은 요청을 8080번으로 전달할 때
# Nginx 설정 예시
server {
    listen 80; # 외부에서 접속하는 문
    location / {
        proxy_pass http://localhost:8080;
    }
}
  1. Spring Boot @Transactional (Dynamic Proxy)
// 내부 동작 원리 (AOP)
1. 클라이언트가 join() 호출
2. 실제 객체가 아닌 '가짜 프록시 객체'가 가로챔
3. 프록시: "DB 트랜잭션 시작" (Begin)
4. 실제 객체의 join() 실행 (Target 로직)
5. 프록시: "성공했으니 DB에 반영" (Commit)

정리​

Proxy는 중간자이다. 복잡하게 생각 할 필요가 없다. 심부름 센터 직원이라고 비유하면 좋을 것 같다.

  • 크롤링: 저 대신 물건 좀 가져다주세요.
  • Nginx: 물건을 분류해서 알맞은 창고로 가져다 주세요.
  • Spring Boot:
    • 오늘 B업체에 돈 가져다주세요.
    • B업체 문 닫혀 있다면 가져간 돈 다시 가지고 돌아오세요.
    • B업체에 돈 가져다주셨으면 장부에 기록해 주세요.
